# 📂 Img/ - 器物展示圖片庫

## 📝 資料夾概述

`Img/` 是一個 **靜態資源目錄**，專門存放所有考古器物的原始照片檔案（`.jpg`）。

當使用者在介面上點擊「擷取物件照片」時，系統會讀取此資料夾中對應的圖片檔案，將其轉換為 Base64 編碼，並顯示在右側的「器物資訊卡」HTML 中，供使用者參考器物的真實樣貌。

## 📏 檔案命名規則 (Naming Convention)

為了讓程式能自動找到正確的圖片，此資料夾內的檔案命名必須**嚴格遵守**以下規則：

1. **格式**：必須為 `.jpg` 檔。
2. **檔名**：必須與 `UtensilsData.json` 中的 **`"器物名稱"`** 欄位完全一致。

### 對照範例

| UtensilsData.json (`器物名稱`) | 對應圖片檔名 |
| --- | --- |
| `"牛罵頭-三連杯陶質基座"` | `牛罵頭-三連杯陶質基座.jpg` |
| `"清水中社-陶缽"` | `清水中社-陶缽.jpg` |
| `"清水中社-圓腹罐"` | `清水中社-圓腹罐.jpg` |

## 🔄 運作流程 (Technical Flow)

此資料夾的運作依賴 `fun.py` 中的 `Tool` 類別：

1. **名稱獲取**：程式先從 `UtensilsData.json` 獲取當前器物的名稱（例如 "陶蓋"）。
2. **路徑組裝**：程式自動組合成路徑字串：`./Img/陶蓋.jpg`。
3. **Base64 編碼**：`Tool.CompileImage()` 函式讀取該 JPG 檔，並將其轉碼為 Base64 字串。
4. **HTML 嵌入**：將編碼後的字串填入 `InfomationCard.html` 的 `src` 屬性中，讓圖片能直接顯示於網頁上，無需依賴外部圖床。

## ⚠️ 注意事項與例外處理

* **缺圖處理**：
若程式在 `Img/` 中找不到對應名稱的 `.jpg` 檔案（例如檔案遺失或命名錯誤），系統**不會報錯崩潰**。
`Tool.CompileImage()` 會自動回傳一個 **透明的 1x1 像素 GIF** 代碼。
> **結果**：資訊卡上的圖片區域將會顯示為空白，但程式仍能繼續運作。


* **圖片尺寸**：
雖然程式未嚴格限制圖片尺寸，但為了配合 `style.css` 的排版，建議使用 **正方形 (1:1)** 或 **4:3** 比例的圖片，以獲得最佳的視覺效果。 CSS 會自動將圖片縮放並裁切圓角。
